### 1-4-1 불변값

- 변수, 상수는 변경 가능성으로 구분
  - 변경 가능성의 대상: **변수 영역의 메모리**
    - 한 번 데이터 할당이 된 변수 공간에 다른 데이터를 재할당 가능 여부가 중요
  - 불변성 여부 구분: **데이터 영역의 메모리**
- 변수(variable) : 바꿀 수 있다
- 상수(constant) : 바꿀 수 없다
- 불변값 ≠ 상수
- 기본형 데이터는 불변값
  - 기본형의 데이터는 데이터 영역에 할당하고, 각 변수는 그 데이터 영역의 주솟값을 가지고 있게 된다.
    ```
    var a = 1;
    var b = 1;
    ```
    결국 변수 a, b 는 같은 주솟값을 바라보고 있는 게 된다. 할당하는 값이 바뀔 때마다, 데이터 영역에서 존재하면 그 주솟값을 재활용하고, 없으면 데이터 영역에 새로운 데이터를 할당하여, 그 주솟값을 바라보게 된다는 것. 결국 **데이터 영역에 있는 값들 자체는 새로운 값을 사용하지 않기 때문에 불변값이라고 볼 수 있다. (불변값의 성질)**
    한 번 만들어진 값을 가비지 컬렉팅을 당하지 않는 한 영원히 변하지 않는다.

### 1-4-2 가변값

- 참조형 데이터는 가변값
  - 하지만 예외도 존재한다.(Object.defineProperty, Object.freeze)
  - 불변값으로 활용하는 방안도 존재0
  - 객체의 변수(프로퍼티) 영역이 별도로 존재
    - 별대로 할애한 영역 → 변수 영역
    - 그러나 변수 영역에는 다른 값을 얼마든지 대입 가능
    - 객체의 경우, 여러개의 값을 가지고 있기 때문에, 데이터를 바로 넣을 수 없다 → 한 단계를 더 거침
    - 다른 데이터영역을 넓게 확보한다.
- 중첩 객체
  - 참조형 데이터의 프로퍼티에 다시 참조형 데이터를 할당하는 경우

```jsx
 컴퓨터의 변수 처리 방식
- 선언과 할당을 나눠서 처리를 하게 된다.

let obj = {
	a:1,
	b:'object'
}

여도, 실제 컴퓨터에서 처리를 할때는 선언을 먼저처리하고, 할당을 하게된다.

let obj;
obj = {
	a:1,
	b:'object'
}
```

- 값을 직접 저장
  - 데이터 할당시에는 빠름
  - 비교에 비용이 많이 듦
  - 메모리 낭비가 심함
- 값의 주소를 저장
  - 데이터 할당시에는 느림
  - 비교에 비용이 들지 않음
    - 같은 값이 오직 하나만 존재 → 불변값!
  - 메모리 낭비 최소화 (데이터 용량이 클수록)

### 1-4-3 변수 복사 비교

- 변수에 할당하기 위해서는 주솟값을 복사해야 한다.
- 엄밀히 따지면 자바스크립트의 모든 데이터 타입은 참조형 데이터
- 기본
  - 주솟값을 복사하는 과정이 한 번
- 참조형
  - 주솟값을 복사하는 과정이 한 단계 더 있다
- 참조형 데이터가 가변값이다 라는 것은
  - 내부의 프로퍼티를 변경할 때만 성립
  - 참조형 데이터 자체를 변경할 경우는 해당 없음
